문제 정의

왜 이 프로젝트인가

핵심 문제

로봇 정책 학습은 대규모 시연 데이터를 필요로 하지만, 실제 로봇 시연은 시간·비용 집약적이다.

Cosmos Policy는 50개 시연으로 SOTA(98.5%)를 달성했지만, 그 모델을 새 로봇에 적응시키는 것 자체가 문제다.

핵심 질문

"50회 시연만으로 수백~수천 회 학습 효과를 낼 수 있는가?"

접근법: 합성 데이터 증강 파이프라인

실제 시연 50+ 회 수집
Cosmos Predict (세계 모델) → 합성 비디오 생성
GR00T IDM (관찰자) → pseudo action label
GR00T N1.6 VLA (두뇌) → 파인튜닝

* 이것이 "이론"이다. 실제는 다음 슬라이드에서 전개된다.

프로젝트의 핵심 동기: 로봇 데이터 수집 비용 문제. 사람이 직접 로봇을 조작해서 데이터를 모으는 것은 매우 느리고 비싸다. Cosmos Policy가 50개 에피소드로 LIBERO 98.5%를 달성한 것은 인상적이지만, 그 후훈련에도 8x H100이 필요하다. 우리의 목표는 RTX 5090 단일 GPU로 OMX 로봇에 맞는 파이프라인을 구축하는 것이다. 세계 모델로 합성 비디오를 만들고, IDM으로 행동을 역추정해서 레이블을 자동 생성하는 접근법이 이론적으로 매력적이다. 이 발표는 프로젝트의 배경, 사전 조사 결과, 그리고 실행 전략을 공유한다.

FK → IK 변환으로 관절값은 OMX 형식으로 변환했으나, 영상 자체는 SO-100 외형이다.

IDM v3 현재 상태

HF: Yuseok/IDM-OMX-v3-50K — pseudo-label 생성 도구로 활용 예정

Normalized MAE: 0.3551 (baseline 대비 -41.7%)

중요: SO-100과 OMX의 관계를 명확히. SO-100은 ROBOTIS가 만든 로봇이지만 OMX와 외형이 다르다. 관절 구성이 유사하여 FK 변환으로 관절 각도를 OMX 좌표계로 바꿀 수 있지만, 영상 자체는 여전히 SO-100 로봇이 찍힌 것이다. 이것이 Cosmos Predict의 OOD 실패 원인 중 하나다. Cosmos가 "OMX 외형"을 본 적이 없기 때문이다. SO-101은 SO-100의 업그레이드 버전으로, FK 기준점이 다르다 (SO-100: 123/116/98/54/-52도 오프셋, SO-101: 0). 558 에피소드 = 원래 158 OMX 에피소드 + SO-101 megamix 400 에피소드를 변환한 것.

사전 조사: IDM 기술 상세

IDM 아키텍처 및 기술 상세

아키텍처 (612M params)

SiGLIP2 (768d) → VL transformer (4L) → DiT (8L, 1024d)

입력: 비디오 2-프레임 쌍 (t, t+1), uint8 256×256

출력: delta actions (rad), 6 dims

OMX Action Space

델타 행동 (rad), ±3°, 6차원

q99 정규화: norm = 2*(x-q01)/(q99-q01) - 1

주요 코드 수정 사항

denormalize NaN 방지: q99==q01 시 division by zero 방지

AV1 → H.264 재인코딩: decord 호환성 문제 해결

embodiment tag 검증: 잘못된 tag 입력 시 조기 에러 + 유효값 안내

CLI 추가: --embodiment-tag 파라미터 명시적 지정

IDM ↔ VLA 관계

가중치 전이 없음 (NO weight transfer)

IDM 출력 → denormalize → Parquet action → VLA 학습 데이터

Cosmos Predict 2.5: 사전 타당성 검토

실험 설정 VRAM 결과 상태
Base V2W 432×768, 93fr, guidance=7 ~20 GB 품질 낮음. post-training 없는 범용 모델 한계 WARNING
Action-Cond (Bridge) 256×320, 37~49fr ~20 GB 정상 동작. 학습 도메인 일치 OK
Action-Cond (OMX) FK 변환, scaler=30.6 ~20 GB 노이즈 심함. 전혀 다른 영상 생성 (OOD) FAIL
Action-Cond v2 guidance=3, steps=50 ~20 GB 여전히 실패. 파라미터 튜닝으로 OOD 해결 불가 FAIL
이 실험들은 프로젝트 착수 전 사전 타당성 검토 차원에서 수행했다. 각 실험의 동기: Bridge 샘플 테스트는 "모델 자체가 제대로 작동하는지" 확인하기 위한 기준선. Base V2W는 합성 데이터 생성 시나리오 확인. OMX 변환 시도는 실제 목표 시나리오. action_scaler 재보정은 FK 변환 후 Cosmos 내부 정규화 범위 문제. OOD 실패는 모델이 OMX 로봇의 외형을 한 번도 본 적이 없기 때문이며, 파라미터 조정으로는 해결 불가. 이것이 post-training이 필수인 이유다.

Cosmos Post-training

⇅ 의존
OMX 영상을 늘리려고
Cosmos를 쓰려는 것

현재 데이터 현황

보유: SO-100 영상 — SO-100 로봇이 찍힌 것

실제 OMX 로봇 데이터: 0 에피소드

Bridge 원본 ~30,000 ep (추정) 대비, post-training 최소 50~100 ep 필요

순환 의존의 핵심: Cosmos 세계 모델이 OMX 로봇의 외형과 동작을 학습하려면 실제 OMX 영상이 필요하다. 그런데 우리가 OMX 영상을 늘리기 위해 Cosmos를 쓰려는 것이다. 이 모순은 근본적이다. SO-100 영상을 FK 변환해서 OMX 관절값으로 바꿔도, Cosmos는 "SO-100 로봇이 찍힌 영상"을 보고 "OMX 로봇"을 생성하라는 요청을 받는다. 그 로봇을 한 번도 본 적이 없으니 노이즈만 생성하는 것이다. Bridge dataset은 30,000 에피소드로 훈련됐다. 최소 50-100이 필요하다고 해도 우리는 현재 0 에피소드(실제 OMX 기준)이다.
Cosmos Transfer 2.5

Cosmos Transfer 2.5: 시각 증강 도구

핵심 개념

원본 비디오의 구조(모션, 레이아웃)는 보존하면서 외관(조명, 재질, 배경)만 변환하는 ControlNet 기반 video-to-video 모델

action/state는 원본 그대로 유지 → 정합성 문제 없음

제어 입력 (Control)

Edge: 윤곽선 보존

Depth: 전후 관계 보존

Seg: 객체별 영역 보존

Blur (Vis): 시각 참조

기대 효과

조명 변화에 대한 정책 강건성 향상

배경/테이블 재질 변화에 대한 일반화

카메라 톤/노출 변화에 대한 내성

X-Mobility 사례: Transfer1로 성공률 54% → 91%

운영 전략

clip 단위 2~6초 분할 증강

원본:증강 비율 = 1:0.25 → 최대 1:0.5

seg IoU 기반 자동 검수 후 통과본만 사용

공식 요구 VRAM: 65.4 GB (48GB+ GPU 필요)

Cosmos Transfer는 Predict와 달리 "새 비디오를 생성"하는 게 아니라 "기존 비디오의 외관만 변환"한다. 따라서 post-training 없이도 범용 모델로 사용 가능하다. X-Mobility 논문에서 Transfer1을 사용하여 visual augmentation으로 성공률을 54%에서 91%로 올린 사례가 있다. 운영 시 주의: 커뮤니티 보고에 따르면 조명만 바꿔도 geometry drift 발생 가능, seg mask를 반드시 함께 사용해야 구조 보존이 안정적이다. 공식 요구 VRAM 65.4GB이므로 A6000(48GB) 이상 GPU가 필요하다.

핵심 고려사항

사전 조사에서 파악한 핵심 고려사항

01 Post-training은 필수 — 범용 세계 모델은 새 로봇 도메인에서 즉시 사용 불가. OOD 문제는 파라미터 튜닝으로 해결 불가능하다.
02 순환 의존 함정 — 데이터를 늘리려면 데이터가 필요. SO-100 영상 ≠ OMX 영상. 실제 로봇 시연 수집이 모든 것의 선행 조건이다.
03 VRAM이 경계를 결정 — 32GB로 할 수 있는 것(SAM3, Predict)과 없는 것(Transfer 65.4GB)이 명확하다. 아키텍처 선택은 하드웨어에 종속된다.
04 경량 대안의 실질적 가치 — SAM3 (3.82 GB)로 세그멘테이션 라벨링을 자동화하고, Cosmos Transfer (48GB+ GPU)와 조합하면 시각 증강 파이프라인이 완성된다.
05 IDM 품질은 꾸준히 개선 가능 — 데이터 확장(158 → 558 ep)과 장기 학습(10K → 50K steps)으로 normalized MAE baseline 대비 -41.7% 달성.
06 이론과 실제의 차이 — 파이프라인 설계 단계에서는 보이지 않는 제약이 실제 실행 시 드러난다. 직접 돌려봐야 안다.
